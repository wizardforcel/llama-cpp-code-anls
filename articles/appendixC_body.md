## C.1 量化类型概览

### C.1.1 什么是量化

量化是将模型权重从高精度浮点数（如FP32）转换为低精度表示（如4位整数）的过程，目的是：

1. **减少模型体积**：4位量化可将模型大小压缩约75%
2. **降低内存带宽**：减少数据传输量，提高推理速度
3. **降低功耗**：边缘设备上更高效

```
┌─────────────────────────────────────────────────────────────────┐
│                      量化过程示意                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   FP32 权重        量化          存储          反量化        计算 │
│   [32位]    ───────────→   [4位]   ───────────→   [FP16/FP32]  │
│                                                                 │
│   0.003921       ───→        1        ───→       0.003906      │
│   0.007843       ───→        2        ───→       0.007812      │
│   0.011765       ───→        3        ───→       0.011719      │
│      ...              ...            ...                        │
│                                                                 │
│   原始大小: 4 bytes/weight                                     │
│   量化后:   0.5 bytes/weight (Q4)                              │
│   压缩比:   8:1                                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## C.2 量化类型详解

### C.2.1 浮点类型

**F32 (GGML_TYPE_F32)**

```c
// ggml/include/ggml.h:220
GGML_TYPE_F32 = 0
```

- **位宽**: 32位 (1× float32)
- **压缩比**: 1:1 (基准)
- **用途**: 原始模型格式，最高精度
- **适用场景**: 训练、精度敏感推理

**F16 (GGML_TYPE_F16)**

```c
GGML_TYPE_F16 = 1
```

- **位宽**: 16位 (1× float16)
- **压缩比**: 2:1
- **精度损失**: 极小（约0.1%困惑度增加）
- **用途**: 推理标准格式
- **适用场景**: GPU推理，精度要求高的场景

**BF16 (GGML_TYPE_BF16)**

```c
GGML_TYPE_BF16 = 30
```

- **位宽**: 16位
- **特点**: 8位指数（与FP32相同），7位尾数
- **优点**: 更宽的动态范围，不易上溢/下溢
- **适用场景**: 训练、混合精度推理

### C.2.2 标准量化 (Q4/Q5/Q8)

**Q4_0 结构解析：**

```
┌─────────────────────────────────────────────────────────────┐
│  Q4_0 Block (32个权重 = 16字节 + 4字节缩放因子 = 20字节)      │
├─────────────────────────────────────────────────────────────┤
│  偏移   │  大小   │  内容                                     │
├─────────┼─────────┼───────────────────────────────────────────┤
│  0      │  4      │  float32 缩放因子 (d)                     │
│  4      │  16     │  32个4位权重 (q[0]..q[31])                │
│         │         │  每个字节存储2个权重:                      │
│         │         │  q[i] = (int4_t)((x[i]/d + 8))            │
├─────────┼─────────┼───────────────────────────────────────────┤
│  总计   │  20     │  每权重: 5 bits (理论4位 + 缩放因子开销)    │
└─────────────────────────────────────────────────────────────┘
```

**特点:**
- 对称量化（零点在8）
- 适合近似对称分布的权重
- 解压公式: `x[i] = d * (q[i] - 8)`

**Q4_1 结构解析：**

```
┌─────────────────────────────────────────────────────────────┐
│  Q4_1 Block (32个权重 = 16字节 + 8字节 = 24字节)              │
├─────────────────────────────────────────────────────────────┤
│  偏移   │  大小   │  内容                                     │
├─────────┼─────────┼───────────────────────────────────────────┤
│  0      │  4      │  float32 缩放因子 (d)                     │
│  4      │  4      │  float32 最小值 (m)                       │
│  8      │  16     │  32个4位权重                              │
├─────────┼─────────┼───────────────────────────────────────────┤
│  总计   │  24     │  每权重: 6 bits                           │
└─────────────────────────────────────────────────────────────┘
```

**特点:**
- 非对称量化
- 解压公式: `x[i] = d * q[i] + m`
- 更适合非对称分布的权重

**Q8_0：**

```c
GGML_TYPE_Q8_0 = 8
```

- 8位对称量化
- 每块32个权重
- 压缩比: 4:1
- 精度损失极小

### C.2.3 K-quant 系列

K-quant使用混合精度策略，对模型中不同重要性的层使用不同精度：

```
┌─────────────────────────────────────────────────────────────┐
│                    K-quant 混合精度策略                       │
├─────────────────────────────────────────────────────────────┤
│  层类型              │  精度          │  原因                │
├──────────────────────┼────────────────┼──────────────────────┤
│  attention.wq/wk/wv  │  较高(F16/Q6)  │  对注意力计算敏感     │
│  attention.wo        │  较高(F16/Q6)  │  输出投影重要         │
│  ffn.w1/w2/w3        │  目标精度      │  主要计算层           │
│  token_embeddings    │  较高(F16)     │  输入嵌入敏感         │
│  output              │  较高(F16)     │  输出层敏感           │
│  其他                │  目标精度      │  标准量化             │
└─────────────────────────────────────────────────────────────┘
```

**Q4_K (推荐日常使用)**

```c
GGML_TYPE_Q4_K = 12  // llama.h: LLAMA_FTYPE_MOSTLY_Q4_K_M
```

**特点:**
- 4位主精度
- attention张量和嵌入层使用更高精度
- 平衡质量与速度
- 推荐作为默认量化选择

**Q5_K**

```c
GGML_TYPE_Q5_K = 13
```

**特点:**
- 5位主精度
- 接近FP16的质量
- 文件大小约为FP16的65%

**Q6_K**

```c
GGML_TYPE_Q6_K = 14
```

**特点:**
- 6位主精度
- 高质量量化
- 适合对质量要求严格的场景

**Q2_K / Q3_K**

```c
GGML_TYPE_Q2_K = 10
GGML_TYPE_Q3_K = 11
```

**特点:**
- 超低位宽
- 显著压缩，但质量损失较大
- 适合超大模型或资源极度受限场景

### C.2.4 IQ (Improved Quantization) 系列

IQ系列使用重要性矩阵（Importance Matrix）进行更智能的量化：

```
┌─────────────────────────────────────────────────────────────┐
│                IQ量化原理                                      │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  1. 收集重要性矩阵(imatrix)                                   │
│     └── 使用校准数据运行模型                                  │
│     └── 记录每个激活值的重要性                                │
│                                                               │
│  2. 基于重要性进行非均匀量化                                  │
│     └── 重要权重 → 更高精度                                   │
│     └── 次要权重 → 更低精度                                   │
│                                                               │
│  3. 结果：相同位宽下更好的质量                                │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

**IQ4_NL**

```c
GGML_TYPE_IQ4_NL = 20
```

**特点:**
- 4位非线性量化
- 使用查找表(LUT)进行反量化
- 通常优于Q4_K

**IQ3_S / IQ3_M**

```c
GGML_TYPE_IQ3_S = 21
```

**特点:**
- 3位重要性感知量化
- IQ3_M质量接近Q4_K，但体积更小

**IQ2_XXS / IQ2_XS / IQ2_S / IQ2_M**

```c
GGML_TYPE_IQ2_XXS = 16
GGML_TYPE_IQ2_XS  = 17
GGML_TYPE_IQ2_S   = 22
GGML_TYPE_IQ2_M   = 29
```

**特点:**
- 2位超低位宽
- 需要imatrix才能达到可接受质量
- 适合超大模型（70B+）

**IQ1_S / IQ1_M**

```c
GGML_TYPE_IQ1_S = 19
GGML_TYPE_IQ1_M = 29
```

**特点:**
- 1位量化
- 仅适用于超大模型
- 必须配合imatrix使用

### C.2.5 特殊量化类型

**TQ1_0 / TQ2_0 (三元量化)**

```c
GGML_TYPE_TQ1_0 = 34
GGML_TYPE_TQ2_0 = 35
```

**特点:**
- 权重取值为 {-1, 0, 1} (TQ1) 或扩展集合 (TQ2)
- 稀疏性优势
- 适合特定硬件加速

**MXFP4 / NVFP4**

```c
GGML_TYPE_MXFP4 = 39
GGML_TYPE_NVFP4 = 40
```

**特点:**
- 遵循Microscaling格式标准
- 更好的数值稳定性
- 未来硬件可能原生支持

## C.5 量化实践指南

### C.5.1 使用 llama-quantize 工具

```bash
# 基本用法
./llama-quantize input.gguf output.gguf Q4_K_M

# 使用imatrix进行IQ量化
./llama-imatrix -m input.gguf -f calibration.txt -o imatrix.dat
./llama-quantize --imatrix imatrix.dat input.gguf output.gguf IQ4_NL

# 混合量化（不同层不同精度）
./llama-quantize input.gguf output.gguf Q4_K_S

# 仅量化特定张量类型
./llama-quantize --token-embedding-type F16 --output-tensor-type F16 \
    input.gguf output.gguf Q4_K_M
```

### C.5.2 量化参数详解

```bash
# 查看所有支持的量化类型
./llama-quantize --help

# 常用参数:
#   -n <nthreads>        使用nthreads线程

#   --token-embedding-type <type>   嵌入层类型
```


## C.6 量化最佳实践

### C.6.1 选择建议

| 场景 | 推荐类型 | 原因 |
|------|----------|------|
| 日常对话 | Q4_K_M | 平衡质量与速度 |
| 代码生成 | Q5_K_M 或 Q6_K | 代码对精度敏感 |
| 数学推理 | Q6_K 或 F16 | 数学需要高精度 |
| 边缘设备 | Q4_K_S 或 Q3_K | 内存受限 |
| 超大模型(70B+) | IQ4_NL + imatrix | 质量优先 |
| 服务器部署 | Q5_K_M | 质量与性能平衡 |
| 移动端 | Q4_K_S | 资源受限 |

### C.6.2 常见问题

**Q: Q4_0 和 Q4_K 有什么区别？**

A: Q4_0是简单的4位量化，所有层使用相同精度。Q4_K是混合精度，对关键层（如attention）使用更高精度，通常质量更好。

**Q: 什么是imatrix，为什么重要？**

A: imatrix（重要性矩阵）通过校准数据识别哪些权重对模型输出更重要。IQ系列量化利用imatrix对重要权重分配更多精度位，显著提升超低比特量化的质量。

**Q: 量化后模型输出不一致怎么办？**

A: 
1. 使用更高精度的量化类型（如Q5_K代替Q4_K）
2. 确保嵌入层和输出层使用F16
3. 对IQ类型，使用高质量的校准数据生成imatrix

**Q: GPU和CPU量化有区别吗？**

A: 量化格式相同，但某些类型在特定后端优化更好。CUDA后端对所有K-quant和IQ类型都有良好支持。

### C.6.3 性能优化技巧

1. **批处理**: 使用更大的batch size摊销量化/反量化开销
2. **GPU卸载**: 合理设置 `n_gpu_layers` 充分利用GPU
3. **混合精度**: 关键层保持高精度，其他层使用低精度
4. **缓存**: 对重复输入使用KV缓存

## 本课小结

本附录整理了llama.cpp支持的量化类型及其适用场景。浮点量化包括F16和F32，特点是无损失或极低损失，适用于高精度需求场景。标准量化包括Q4_0和Q8_0，特点是简单快速，适用于通用场景。K-quant系列包括Q4_K和Q5_K，采用混合精度策略，是推荐的默认选择。IQ系列包括IQ4_NL和IQ3_M，使用重要性感知技术，适用于超大模型。特殊量化包括TQ和MXFP4，针对特定优化目标，适用于实验或特定硬件场景。

**量化命令速查：**

```bash
# 推荐默认
llama-quantize input.gguf output.gguf Q4_K_M

# 高质量
llama-quantize input.gguf output.gguf Q5_K_M

# 使用imatrix
llama-imatrix -m input.gguf -f calib.txt -o im.dat
llama-quantize --imatrix im.dat input.gguf output.gguf IQ4_NL

# 评估质量
llama-perplexity -m model.gguf -f test.txt
```
